# INEXISTENCE 设计规范

> 版本：1.0 · 适用于当前 Astro 7 博客
> 规范依据：项目 README、现有页面与 `src/styles/global.css`、`animal-island-ui` 1.3.0 设计文档
> 本文是新增页面、组件和视觉调整的设计基准。现有代码与本文冲突时，应先判断代码是有意例外还是历史遗留，再统一实现与文档。

## 1. 设计定位

INEXISTENCE 是一座持续生长的个人记忆岛屿。它保存技术笔记、生活碎片和旧事，核心体验不是“浏览产品”，而是“沿小径散步、偶遇一篇文章”。

视觉关键词：

- 温暖、安静、自然、略带游戏感；
- 手作纸张、圆润轮廓、岛屿路标和轻微漂浮；
- 内容优先，不用装饰压过标题、摘要和正文；
- 怀旧但不复古，亲切但不幼稚。

设计继承 `animal-island-ui` 的暖纸背景、胶囊控件、圆角卡片、NookPhone 色彩和按压反馈，但不复制其文档站布局。本站通过更克制的配色、更大的留白和“岛屿日志”叙事形成独立识别。

## 2. 核心原则

### 2.1 像散步，不像信息流

首页文章以弯曲小径和站点排列；归档以时间小径组织；文章页提供“沿途路标”。新增信息架构应优先采用旅程、地点、路标、日志等隐喻，避免仪表盘式密集堆叠。

### 2.2 先阅读，后装饰

插画只承担欢迎、分区、引导或节奏分隔。正文区域保持安静，单个视口内不同时使用多个高饱和插画、纹理和强动画。

### 2.3 圆润而有层次

交互元素不使用尖锐直角。层次主要通过色块、边框、纹理和少量底部阴影建立，不使用通用 SaaS 风格的大面积模糊悬浮阴影。

### 2.4 动效表达生命感

动效幅度小、速度温和，只用于进入、悬停、按压和方向提示。所有非必要动画必须尊重 `prefers-reduced-motion`。

### 2.5 中文阅读优先

中文正文的行高、栏宽和段落节奏优先于英文展示字效果。英文只用于品牌、眉题和少量岛屿标识。

## 4. 字体与排版

### 4.1 字体角色

- 英文品牌、数字、眉题、展示标题：`Nunito`，回退 `sans-serif`；
- 中文正文与界面：`Noto Sans SC`, `PingFang SC`, `sans-serif`；
- 组件库已随样式打包 Nunito 与 Noto Sans SC 字体文件，不再重复引入远程字体。

### 4.2 字重

- 正文：400–500；
- 导航、按钮、标签：600–700；
- 中文标题：700–800；
- 英文品牌与超大标题：800–900；
- 日期数字或短标识可用 800–900，不用于长段文字。

### 4.3 字号与行高

| 角色 | 建议值 | 规则 |
| --- | --- | --- |
| 首页品牌标题 | `clamp()`，约 64–123px | Nunito 900，紧凑行高与负字距 |
| 页面 H1 | `clamp()`，约 40–72px | 一至两行，移动端避免单字孤行 |
| 分区 H2 | `clamp()`，约 32–56px | 可配英文眉题 |
| 卡片 H3 | 22–30px | 最多两行 |
| 正文 | 16–18px | 行高 1.75–1.9 |
| 元数据 | 12–14px | 颜色弱化，不低于 12px |
| 眉题 | 约 12px | Nunito 800，全大写，字距 `0.19em` |

正文舒适栏宽以 65–75 个中文字符以内为目标；文章主体不得铺满 `1120px` 容器。

## 5. 间距、尺寸与布局

### 5.1 基础节奏

以 4px 为最小单位，常用序列为：`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64 / 76 / 96 / 112 / 144`。

- 组件内部优先用 8–32px；
- 卡片内容区通常 20–32px；
- 分区间距使用 `clamp(76px, 10vw, 144px)`；
- 不为“填满空间”压缩段落和卡片之间的呼吸感。

### 5.2 页面容器

- 最大内容宽度：`--shell: 1120px`；
- 桌面水平安全区：20px，即 `width: min(calc(100% - 40px), 1120px)`；
- 移动端安全区可收至 16px，但不得贴边；
- 超出容器的元素仅限背景纹理、场景插画、分隔波浪和有意的满宽图片。

### 5.3 响应式断点

现有样式主要在 `950px / 900px / 760px / 700px / 430px` 调整。新增模块应沿用相邻页面已有断点，不增加只差几十像素的碎片断点。

- `> 900px`：页面场景 Hero 使用完整桌面构图；
- `701–900px`：页面场景 Hero 收缩插画、减少横向偏移；
- `≤ 760px`：单栏首页与简化路线；
- `≤ 700px`：归档切换为 Collapse，文章目录退出侧栏；
- `≤ 430px`：收紧字号、按钮和装饰物，保持 320px 最小视口可用。

## 6. 形状、图形与图像

### 6.1 圆角

- 胶囊导航、按钮：`999px` 或组件库的完整胶囊圆角；
- 默认卡片：`20px`；
- 大容器：`30px` 左右；
- Collapse：约 `18px`；
- 图片：跟随所属卡片圆角，常用 18–30px；
- 交互元素最小建议圆角 12px，除非它是圆形路标或头像。

### 6.2 纹理

纹理采用低对比叶片、圆点、纸张或波浪图案，透明度通常低于 20%。同一分区最多一个主纹理层；正文背景不铺高频纹理。

组件库 `Card.pattern` 表示壁纸感：两层稀疏圆点、浅色底和 1.5px 同色边框。首要卡片可用 pattern，普通连续列表应克制使用。

### 6.3 插画和素材

- 优先复用 `public/assets/animal-island-ui` 和项目已有岛屿素材；
- 文章封面和正文图片放在 `public/assets/blog-images/<文章名或主题>/`，不要混入页面场景目录；
- 页面专属的 Hero 和装饰场景放在 `public/assets/page-scenes/<页面名>/`，不要放入 `blog-images`；
- Waline 匿名留言头像池放在 `public/assets/avatars/`；增删或重命名时必须同步 `waline-for-blog/index.cjs` 的 `avatarFiles`，并先部署博客静态资源再部署 Waline；
- `public/assets/animal-island-ui/` 只存放本站直接复用的组件库配套静态素材，保留原许可与署名要求，不接收文章或单页专属图片；
- 只有跨页面复用且无法归入组件库素材的通用图片才放在 `public/assets/images/`；
- 素材承担明确语义：动物欢迎、路标定位、分类入口、页尾场景；
- 装饰图片使用空 `alt` 并设置 `aria-hidden="true"`；有内容意义的图片写具体中文替代文本；
- 文章封面使用 `object-fit: cover`，不得拉伸；
- 新素材需保持柔和色彩、圆润轮廓和轻微纸艺/游戏 UI 质感，避免写实 3D、霓虹和玻璃拟态。

### 6.4 插图生成规范

本节适用于通过图像生成模型制作的新角色、物件、场景、文章封面和纹理。角色、场景语言和通用物件统一采用《集合啦！动物森友会》的岛屿生活视觉体系，使访问者一眼进入同一个世界。页面叙事与构图服务于 INEXISTENCE，但不要求把场景和物件改造成另一套原创画风。允许使用游戏中的既有角色、自然环境、建筑语汇、家具类型和生活道具；不得直接提取或逐像素复刻游戏贴图、截图、UI 和官方宣传画面。

#### 6.4.1 视觉风格

统一采用《集合啦！动物森友会》式的“柔和微缩岛屿 / 圆润游戏插图”风格，并按用途分为两种表现：

- **场景模式**：微缩岛屿模型感，圆润低多边形造型，轻微俯视或近似正交视角，柔和自然日光，简洁材质和舒适生活气息；
- **剪影模式**：角色与单个物件从场景中独立出来，保留同样的圆润三维或柔和游戏渲染质感，用透明背景交付，不改成另一套扁平图标风格。

两种模式共享以下特征：

- 造型由大而圆的基础形组成，轮廓清楚，缩小后仍能辨认；
- 视角以正面、轻微俯视或平缓等距视角为主，不使用强透视和广角畸变；
- 材质像游戏中的木材、布料、草地、黏土和烤漆家具，细节简化但有柔和体积；
- 明暗关系简单，使用自然环境光、柔软接触阴影和克制高光，不做电影级体积光；
- 细节集中在能表达身份的 1–3 个特征，避免高密度纹样、毛发和复杂材质；
- 气质友善、从容、略带幽默感，保持游戏中轻松日常、季节变化与岛屿生活的氛围；
- 插图中不得自动生成文字、Logo、水印、二维码或难以辨认的符号。

现有 `animal-icon.webp` 可作为比例与质感参照：头部和五官偏大、身体紧凑、四肢短、服装图案简化、边缘柔和。生成指定《动物森友会》角色时，应保留其可识别的物种、脸部花纹、标志色和关键服装特征；姿势、道具、背景和画面叙事应为本站重新创作。

#### 6.4.2 色彩控制

- 主色优先从 `--island-mint`、`--island-cream`、`--island-sand`、`--island-brown` 和 NookPhone 功能色中选择；
- 单张插图以 1 个主色、1–2 个辅助色、2–3 个中性色为宜；
- 整体中等偏低饱和，最鲜艳颜色只用于面积小于约 15% 的视觉焦点；
- 深色不用纯黑，优先使用深棕或深薄荷绿；浅色不用纯白，优先使用奶油白或暖纸色；
- 场景插图应预留与页面文字足够对比的安静区域，不能在文字背后放置高频细节；
- 禁止霓虹紫蓝、赛博朋克渐变、重 HDR、玻璃拟态和大面积冷灰。

#### 6.4.3 构图类别与交付尺寸

| 类型 | 推荐画布 | 构图与安全区 | 输出 |
| --- | --- | --- | --- |
| Hero 场景 | 1600×1000 或更高 | 主体放在预定一侧；至少 35% 留作标题区；关键内容离边缘 8% | WebP，必要时透明 PNG |
| 页面头图/路线场景 | 1440×560 或 2 倍图 | 横向延展；上下可裁切；文字区低细节 | WebP |
| 角色组合 | 1600×1200 或更高 | 全身完整，脚下留 6–10% 空间，不切耳朵、手和道具 | 透明 PNG/WebP |
| 单个物件/路标 | 512×512 或 1024×1024 | 主体占画布 70–82%，四周留均匀透明边距 | 透明 PNG/WebP |
| 文章封面 | 1600×900（16:9） | 焦点不要贴边，兼容居中裁切；禁止把标题画进图片 | WebP/JPEG |
| 背景纹理 | 512×512 无缝平铺 | 低对比、无明显中心、接缝不可见 | WebP |

生成模型输出是位图时，不伪装成 SVG。只有经过人工重绘、路径清理并验证的矢量素材才保存为 SVG。

#### 6.4.4 透明背景与边缘

- 独立角色和物件默认要求真实透明背景，不接受棋盘格被烘焙进图像；
- 边缘不应出现白边、黑边、残留底色、半透明噪点或大面积外发光；
- 允许 1–3px 的柔和暖色描边帮助小尺寸识别，但同组素材必须一致；
- 投影若属于物件本身，应保留为低透明软阴影；若由页面布局控制，则源文件不带投影；
- 透明素材导出后必须分别在奶油色、薄荷绿和深棕色背景上检查边缘。

#### 6.4.5 角色设计约束

- 角色分为两类：`AC 既有角色` 与 `本站原创岛民`。生成前必须明确所属类型，不能让原创角色无意间变成某个既有角色的近似替代；
- `AC 既有角色` 必须写出准确的中/英文角色名，并固定物种、脸部花纹、耳朵/尾巴形状、标志色和常见服装特征；
- `本站原创岛民` 使用原创物种、服装、五官和配色组合，不照搬某个既有角色的辨识特征；
- 头身比建议约 1:1 至 1.5:1，手脚短而清晰，轮廓在 96px 高度仍可辨认；
- 表情以欢迎、专注、散步、阅读、回忆等温和状态为主；
- 手持物最多 1 件，必须与场景叙事有关，如信件、书、路牌、叶片或工具；
- 同一角色系列固定头形、耳形、眼距、脸部花纹、标志色和服装结构，只改变姿势与小道具；
- 不混合两个角色的五官或花纹，不随意改变角色物种、年龄感和核心性格；
- 可借鉴游戏中的角色设定，但不重画官方立绘、截图构图、活动海报或带版权标识的素材。

#### 6.4.6 场景设计约束

- 每张场景只表达一个主要事件：欢迎登岛、沿路阅读、整理岛志或在码头告别；
- 场景语言保持《集合啦！动物森友会》的微缩岛屿感：圆润地形、弧形河岸、整洁草地、玩具般建筑、季节植被和舒适生活设施；
- 采用前景点缀、中景角色/事件、背景环境三层即可，不堆叠过多景深层；
- 建筑、树木、路牌、桥梁、围栏、灯具和家具应保持游戏式圆角比例、简化材质与软阴影，避免照片级结构精度；
- 通用物件可以直接采用游戏中的物件类别和造型语言，如叶子图标、铃钱袋、星星棒、木制路牌、气球礼物、化石、铲子、钓竿、DIY 工作台和 NookPhone 应用图标；
- 使用具体游戏物件时，在生成记录中写明物件名称；不得从游戏资源包中直接提取贴图或模型；
- 植被和道具可重复形成游戏式陈列节奏，但形状、尺寸与角度需有轻微变化；
- 场景季节、天气和时间必须统一，不在同一画面混用雪地、夏季阔叶树和樱花等冲突状态；
- 页面会叠加的年份、标题、按钮和路径节点由 HTML/CSS/SVG 绘制，不生成进背景图；
- 重要路径、角色和道具不能放在响应式裁切高风险区。

#### 6.4.9 生成、筛选与后处理流程

1. 先用同一提示词生成 3–4 个构图候选，只评估轮廓、留白和视觉重心；
2. 选定构图后固定主体描述和配色，再迭代表情、道具与细节；
3. 检查手脚、五官、重复物、透视、文字伪影和透明边缘；
4. 必要时局部重绘，不用整体锐化掩盖结构错误；
5. 裁切、去底、清边并转换到 sRGB；
6. 保留高分辨率母版，再生成网站交付版本；
7. 在真实页面的桌面、平板和 320px 移动端预览后才可合入。

禁止把未经筛选的首张生成结果直接放入生产目录。

#### 6.4.10 文件与性能

- 文件名使用小写英文与连字符，如 `archive-trail-summer-v1.webp`；
- 版本变化影响构图时递增 `v2`，仅压缩变化不新建视觉版本；
- 母版与生成记录不要放入站点公开资源目录；生产文件按用途放在 `public/assets/blog-images/<文章名或主题>/`、`public/assets/page-scenes/<页面名>/` 或其他既有分类目录；
- 透明插图优先 WebP；兼容或边缘质量需要时使用 PNG；照片型文章封面可用 JPEG；
- Hero 和页面头图建议控制在 300KB 以内，单个物件建议 80KB 以内；在画质明显受损时允许例外，但必须记录原因；
- 页面中按实际展示尺寸设置 `width`、`height` 或 `aspect-ratio`，避免布局偏移；非首屏图使用懒加载；

## 7. 组件规范

### 7.1 导航与站点标识

- 页头在视觉首屏上方绝对定位，最大宽度与 `.shell` 一致；
- 品牌标识由叶片图形和 `INEXISTENCE` 组成；
- 主导航采用半透明奶油色胶囊，当前项使用深棕底与浅色字；
- 当前页面必须通过 `aria-current="page"` 表达；
- 新增一级导航前先验证移动端 320px 宽度，避免折行。

### 7.2 按钮

优先使用 `animal-island-ui` 的 `Button`。

- 主行动：`type="primary"`，每个视觉区最多一个；
- 次行动：`type="default"`；
- 大按钮高度约 48px、水平内边距 32px；
- 悬停轻微上移，按下向下回落，不能只靠颜色表示状态；
- 链接跳转优先使用真实 `<a>`。现有以 `form action` 包裹按钮的模式可维护，但新增实现应确保无 JavaScript 时仍可访问；
- 按钮文字采用动作短语，如“开始逛岛”“翻开完整岛志”。

### 7.3 卡片

优先使用 `Card`，并通过外层语义元素补充链接和文章结构。

- 默认背景为暖纸色，圆角 20px；
- 可点击卡片整卡可点，且有清晰的可访问名称；
- 信息顺序：分类/日期 → 标题 → 摘要 → 行动提示；
- 摘要建议 2–4 行，不把完整正文塞入列表卡片；
- Featured 可使用彩色 `pattern`；普通卡片使用 `default` 或 `none`；
- 悬停只上浮约 2px，避免旋转、缩放和强发光。

### 7.4 标签

- 分类使用 `Tag`，颜色来自统一分类映射；
- 列表通常使用 `small`，文章页头可使用 `medium`；
- 彩色页头内用 `solid`，纸张列表内可用 `soft`；
- 标签只显示短分类或状态，不承载句子。

### 7.5 标题与分隔线

- 年份等强调短标题可使用 `Title` 缎带；普通文章标题不要套缎带；
- `Divider` 用于叙事章节切换：黄色波浪表示明快过渡，棕色虚线表示历史/列表分段；
- 不同时叠加普通 `<hr>`、波浪和大留白来表达同一次分隔。

### 7.6 Collapse

- 小屏归档使用 `Collapse`，首个年份默认展开；
- 问题行必须概括年份和文章数；
- 展开动画约 300ms，使用高度网格过渡；
- 内容项保持整行可点击，日期和分类不抢标题层级。

### 7.7 页脚

- `Footer type="tree" seamless` 构成自然场景收尾；
- 下方必须保留站点名、欢迎文案、组件库署名与 CC BY-NC 4.0 许可说明；
- 页脚信息低对比但仍需满足可读性，外链明确可聚焦。

## 8. 页面模式

### 8.1 首页

结构顺序固定为：欢迎 Hero → 文章小径 → 分类目录 → 岛屿留言 → 场景页脚。

- Hero 是全站最具表现力的区域，左右为文字与岛民场景；
- 首页只展示最新 5 篇文章，完整内容导向归档；
- 文章站点左右交错，但 DOM 阅读顺序始终按时间；
- 分类入口使用类似岛上 App 的图标块；
- 移动端将路径简化为单轴，不强行保留桌面的左右摆动。

### 8.2 归档、分类与留言板

- 归档 Hero 使用时间路径插画和年份路标；
- 归档与留言板的场景 Hero 统一采用左侧文案安全区、右侧等尺寸装饰场景和分层进入动效；页面可保留不同背景与插画叙事，但不得另建布局、定位、缩放或断点规则；
- 场景 Hero 的眉题、标题、说明与插画分别进入；插画为装饰性内容，使用空替代文本并从辅助技术中隐藏；
- 场景插画始终完整落在内容安全区内，不得为营造出血效果裁切关键角色或道具；移动端继续保留缩小后的场景并避让文案，在 360px 及以下统一隐藏场景插画，优先保障阅读；
- 桌面按年份展示列表与虚线分隔，移动端切换为 Collapse；
- 分类页保持同一页面 Hero 语法，不为每个分类发明不同结构；
- 日期、标题、分类三者对齐稳定，标题拥有最高视觉权重。

### 8.3 文章页

- 彩色页头颜色与分类一致；
- 页头顺序：返回入口 → 分类/日期/地点 → 标题 → 摘要；
- 正文是页面最安静的区域，使用纸张底、深棕字和充足行高；
- 标题超过两个时显示“沿途路标”目录；桌面可置于侧栏，移动端合并到正文流或隐藏；
- 图片、引用、代码、表格必须在 320px 宽度下不造成页面横向滚动；
- 页尾提供更新/更早文章导航，文案明确时间方向。

### 8.4 404

延续“迷路/回岛”的轻叙事，提供唯一明确的返回行动。错误页不引入独立视觉品牌。

## 9. 正文内容样式

- H2 使用明确的段落间距和小型岛屿标记；H3 层级弱于 H2；
- 正文链接使用可见下划线或明确颜色变化，不只依赖悬停；
- 引用采用暖纸或浅色块、左侧强调和装饰引号，正文仍保持足够对比；
- 行内代码使用浅底圆角；代码块使用深棕黑背景和约 20px 圆角；
- 表格允许横向滚动，表头与单元格边界清晰；
- 内容图片圆角显示；作者显式添加 `.full-width` 时才可突破正文栏宽；
- 空段落、连续多个 H1 和无替代文字的内容图片不应发布。

## 10. 动效

### 10.1 动效层级

- 页面进入：元素由轻微下移和透明进入，约 400–700ms；
- 连续内容：使用 60–90ms 的轻微错峰；
- 控件反馈：约 180–250ms；
- Collapse：约 300ms；
- 环境动效：漂浮、纹理、太阳旋转等可更慢，幅度不超过元素尺寸的 3–6%。

### 10.2 禁止模式

- 正文随滚动持续视差；
- 大幅缩放或旋转卡片；
- 多个装饰物以不同高频节奏同时运动；
- 动画导致内容位置变化、布局抖动或延迟主要操作；
- 忽略 `prefers-reduced-motion`。

## 11. 可访问性

- 页面语言使用 `zh-CN`，每页只有一个主 H1；
- 保留“跳到主要内容”链接；
- 键盘焦点使用 3px 珊瑚色轮廓、4px 偏移；组件库控件可保留自身明确焦点环；
- 正文和重要界面文字以 WCAG AA 为最低目标：普通文字至少 4.5:1，大字至少 3:1；
- 不用颜色作为分类、当前状态或错误的唯一线索；
- 所有交互区域建议至少 44×44px；
- 图标按钮需要可访问名称；纯装饰 SVG/图片不进入辅助技术阅读流；
- 保持自然 DOM 顺序，CSS 交错布局不得改变键盘和屏幕阅读器顺序；
- 站点在 JavaScript 关闭时仍应能够导航和阅读文章。

## 12. 文案语气

文案像岛上的主人招呼来访者：温和、具体、略带画面感，但不过度角色扮演。

推荐词汇：登岛、岛志、小径、路标、散步、码头、旧事、记录、沿途。
避免词汇：赋能、探索无限可能、沉浸式体验、点击这里、震撼、颠覆。

- 按钮用 4–8 个字的动作短语；
- 摘要解释内容本身，不堆叠岛屿隐喻；
- 英文眉题保持短、全大写，作为方向标识而不是主要信息；
- 日期统一由 `formatDate()` 输出，不在组件中自行拼格式。

## 13. 开发约束

- 全局令牌和跨页面规则放在 `src/styles/global.css`；
- 可复用结构放在 `src/components`，页面只组合内容；
- 组件库样式必须在自定义全局样式之前引入：`animal-island-ui/style` → `global.css`；
- 优先从组件具体路径导入，以保持当前 Astro/React 兼容方式；
- 颜色分类映射必须集中管理，禁止在页面模板复制映射；
- 新增 CSS 类沿用 BEM 风格：`.block__element--modifier`；
- 使用 `clamp()` 处理展示字号和大区间距，避免只为单一设备写死数值；
- 不覆盖不相关的组件库内部选择器；必须覆盖时限定在本站块级类之下；
- 本项目和 `animal-island-ui` 仅用于非商业个人博客，保留原作者署名与 CC BY-NC 4.0 许可说明。
